# ob-douyin

这是一个用于归档抖音视频链接、提取视频文案并在 Obsidian 中维护记录的本地知识库。

仓库的核心目标是让每条抖音视频链接、对应文案 Markdown 文件和提取记录保持一一对应，方便后续检索、阅读和二次整理。

## 目录结构

```text
.
├── README.md
├── 未命名.base
└── 抖音视频文案/
    ├── 00抖音视频链接.md
    ├── 01提取记录.md
    ├── 03同步备注.md
    ├── extract.py
    └── *.md
```

## 核心文件

- `抖音视频文案/00抖音视频链接.md`：存放待处理的抖音视频链接及相关描述。
- `抖音视频文案/01提取记录.md`：记录已提取的视频链接、生成的文案文件和提取时间。
- `抖音视频文案/03同步备注.md`：说明链接、记录和文案文件之间的同步规则。
- `抖音视频文案/extract.py`：批量读取新链接、调用转写脚本并生成文案 Markdown。
- `抖音视频文案/*.md`：每个视频对应的一份文案归档。

## 使用方式

1. 将新的抖音视频链接添加到 `抖音视频文案/00抖音视频链接.md`。
2. 进入文案目录：

   ```bash
   cd 抖音视频文案
   ```

3. 运行提取脚本：

   ```bash
   python3 extract.py
   ```

4. 脚本会自动跳过 `01提取记录.md` 中已有记录的链接，只处理新增链接。
5. 提取成功后，脚本会生成对应的视频文案 Markdown，并将结果追加到 `01提取记录.md`。

## 依赖说明

`extract.py` 依赖本机的抖音转写脚本：

```text
~/.claude/skills/douyin-transcript/scripts/transcribe.py
```

如果该脚本不存在或不可用，文案提取会失败。

## 维护约定

- `00抖音视频链接.md`、`01提取记录.md` 和具体文案 Markdown 应保持一一对应。
- 已处理链接不要从 `01提取记录.md` 中随意删除，避免重复提取。
- 建议通过 `extract.py` 生成文案文件，减少文件名、记录链接和提取时间不一致的问题。
- 本仓库包含 Obsidian 配置，适合直接作为 Obsidian vault 打开和维护。
