# ob-douyin

这是一个用于归档抖音视频链接、提取视频文案，并在 Obsidian 中持续维护记录的本地知识库。

仓库的核心目标是让每条抖音视频链接、对应文案 Markdown 文件和提取记录保持一一对应，方便后续检索、阅读和二次整理。

## 目录结构

```text
.
├── README.md
├── 未命名.base
└── 抖音视频文案/
    ├── 00抖音视频链接.md
    ├── 01提取记录.md
    ├── 03同步备注.md
    ├── extract.py
    └── 文案/
        └── *.md
```

## 核心文件

- `抖音视频文案/00抖音视频链接.md`：存放待处理的抖音视频链接及相关描述。
- `抖音视频文案/01提取记录.md`：记录已提取的视频链接、生成的文案文件和提取时间。
- `抖音视频文案/03同步备注.md`：说明链接、记录和文案文件之间的同步规则。
- `抖音视频文案/extract.py`：批量读取新链接、调用转写脚本、生成文案 Markdown，并同步更新提取记录。
- `抖音视频文案/文案/*.md`：每个视频对应的一份文案归档。

## 使用方式

1. 将新的抖音视频链接添加到 `抖音视频文案/00抖音视频链接.md`。
2. 进入文案目录：

   ```bash
   cd 抖音视频文案
   ```

3. 运行提取脚本：

   ```bash
   python3 extract.py
   ```

4. 脚本会自动读取 `00抖音视频链接.md` 中的抖音短链，并跳过 `01提取记录.md` 中已经出现过的链接。
5. 对每条新增链接，脚本会：
   - 调用本机转写脚本获取视频标题和文案；
   - 将标题清理为合法文件名；
   - 在 `文案/` 子目录生成对应 Markdown 文件；
   - 将视频链接、文案文件链接和提取时间追加到 `01提取记录.md`。

## 依赖说明

`extract.py` 的硬依赖是本机抖音转写脚本：

```text
~/.claude/skills/douyin-transcript/scripts/transcribe.py
```

如果该脚本不存在、不可执行，或返回的 JSON 无法解析，对应视频会提取失败。

脚本还支持可选的语义分段增强：如果本机安装了 `dashscope` SDK，并设置了 `DASHSCOPE_API_KEY`，会调用 `qwen-plus` 对转写文本做语义分段。未安装 SDK、未设置 key 或分段调用失败时，脚本会保留原始转写文本，不会中断提取流程。

## 维护约定

- `00抖音视频链接.md`、`01提取记录.md` 和 `文案/` 下的具体文案 Markdown 应保持一一对应。
- 新链接统一追加到 `00抖音视频链接.md`，不要直接手写 `01提取记录.md` 中的新记录。
- 已处理链接不要从 `01提取记录.md` 中随意删除，避免重复提取。
- 建议通过 `extract.py` 生成文案文件，减少文件名、记录链接和提取时间不一致的问题。
- 单条链接失败不会阻止脚本继续处理后续链接；运行结束时可根据终端输出检查失败项。
- 本仓库包含 Obsidian 配置，适合直接作为 Obsidian vault 打开和维护。
